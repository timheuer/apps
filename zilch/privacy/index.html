Privacy Policy

Zilch

Last updated: March 2026

Summary

Zilch is designed with privacy as a core principle. The app:

  • Collects no personal data
  • Has no analytics or telemetry
  • Makes no network requests
  • Stores all data locally on your device

Data Collection

We do not collect any data.

Zilch does not collect, transmit, or share any personal information, usage data, or analytics. The app has no user accounts, no registration, and no external backend.

Data Storage

All data in Zilch is stored locally on your device:

  • Game progress and scores
  • Game settings and preferences

This data never leaves your device.

Third-Party Services

Zilch uses no third-party services:

  • No analytics (no Google Analytics, Firebase, Mixpanel, etc.)
  • No crash reporting services
  • No advertising networks
  • No social media integrations
  • No external APIs

Network Activity

Zilch makes no network requests. The app:

  • Does not phone home to any server
  • Does not check for updates via external endpoints
  • Does not load remote assets or configurations
  • Functions fully offline

Tracking

Zilch does not track you in any way:

  • No device fingerprinting
  • No advertising identifiers
  • No cross-app tracking
  • No user behavior tracking

This is declared in the app's PrivacyInfo.xcprivacy manifest with NSPrivacyTracking set to false and empty tracking domains.

Apple Privacy Nutrition Label

Based on Apple's App Privacy requirements, Zilch declares:

Category Declaration
Data Used to Track You None
Data Linked to You None
Data Not Linked to You None

Permissions

Zilch requests no permissions:

  • No camera access
  • No photo library access
  • No location access
  • No contacts access
  • No notification permissions

Children's Privacy

Zilch does not collect any data and is safe for users of all ages. There is no content that requires age restrictions.

Changes to This Policy

If we make changes to this privacy policy, we will update the "Last updated" date at the top. Since we collect no data and have no way to contact users, significant changes would only be communicated through App Store update notes.
